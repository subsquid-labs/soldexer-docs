In a few minutes, you'll have a running pipe that indexes Orca Whirlpool swap instructions on Solana mainnet into a local PostgreSQL database.

## Prerequisites

- Node.js 22.15+
- A package manager — pnpm, npm, yarn, or bun. Snippets below assume `pnpm` (`npm install -g pnpm`); substitute as needed.
- Docker (for the bundled PostgreSQL container)

## Initialize the project

Run the CLI in the directory where you want the project folder to land:

```bash
pnpx @subsquid/pipes-cli@1.0.0-alpha.4 init
```

The CLI prompts for the project folder name, package manager, sink, network type, network, and template; then installs dependencies and writes a runnable project.

For a non-interactive run — useful when scripting or pasting from docs — pass `--config` with a JSON string. The configuration below matches the description at the top of this page:

```bash
pnpx @subsquid/pipes-cli@1.0.0-alpha.4 init --config '{
  "projectFolder": "orca-example",
  "packageManager": "pnpm",
  "sink": "postgresql",
  "networkType": "svm",
  "network": "solana-mainnet",
  "templates": [
    {
      "templateId": "custom",
      "params": {
        "contracts": [
          {
            "contractAddress": "whirLbMiicVdio4qvUfM5KAg6Ct8VwpYzGff3uctyCc",
            "contractName": "OrcaWhirlpool",
            "contractEvents": [
              {
                "name": "swap",
                "type": "instruction",
                "inputs": [
                  { "name": "amount", "type": "u64" },
                  { "name": "sqrtPriceLimit", "type": "u128" }
                ]
              }
            ],
            "range": { "from": "latest" }
          }
        ]
      }
    }
  ]
}'
```


`--config` also accepts a path to a JSON file. To inspect the full config schema (all sinks, networks, and template params), run `pipes init --schema`.

<Note>
`sink` currently accepts `"postgresql"` and `"clickhouse"`. The schema also lists `"memory"`, but it isn't wired up yet — the CLI errors out at the `write-index-ts` stage if you try.
</Note>

## Run the pipeline

The generated project ships with a `docker-compose.yml` that brings up Postgres and the pipeline together:

```bash
cd orca-example
docker compose --profile with-pipeline up
```


For an iterative dev loop, run Postgres in Docker and the pipeline locally:

```bash
docker compose up -d         # Postgres on :5432
pnpm run db:migrate          # apply the generated migration
pnpm run dev                 # tsx src/index.ts
```

Either way, rows start landing in the `orca_whirlpool_swap` table within a minute.

## What was generated

The project layout:

```
orca-example/
├── src/
│   ├── index.ts        # the pipe — source, decoder, target
│   ├── schemas.ts      # Drizzle table definitions
│   ├── contracts/      # generated ABI bindings (per program ID)
│   └── utils/
├── migrations/         # SQL migrations generated by drizzle-kit
├── docker-compose.yml  # Postgres + optional pipeline service
├── Dockerfile
├── drizzle.config.ts
├── package.json
├── .env                # DB_CONNECTION_STR — points at local Postgres
└── README.md
```

The pipe lives in `src/index.ts`. The decoder block defines what to extract:

```ts
const custom = solanaInstructionDecoder({
  range: { from: 'latest' },
  programId: ['whirLbMiicVdio4qvUfM5KAg6Ct8VwpYzGff3uctyCc'],
  instructions: {
    swap: orcaWhirlpoolInstructions.swap,
  },
}).pipe(enrichEvents)
```

The decoder asks the Portal for `swap` instructions on the Orca Whirlpool program. `enrichEvents` (from `src/utils/`) reshapes each decoded instruction into a row matching the Drizzle table. See [solanaInstructionDecoder](./reference/utility-components/instruction-decoder) and [Transformer](./reference/basic-components/transformer).


The `main()` function wires the decoder to a [drizzleTarget](./reference/basic-components/target/drizzle-target):

```ts
export async function main() {
  await solanaPortalSource({
    id: 'd59cd591',
    portal: 'https://portal.sqd.dev/datasets/solana-mainnet',
    outputs: { custom },
  }).pipeTo(
    drizzleTarget({
      db: drizzle(env.DB_CONNECTION_STR),
      tables: [orcaWhirlpoolSwapTable],
      onData: async ({ tx, data }) => {
        for (const values of chunk(data.custom.swap)) {
          await tx.insert(orcaWhirlpoolSwapTable).values(values)
        }
      },
    }),
  )
}
```


The `id` is a random per-project identifier — keep it stable so the [target's cursor](./guides/architecture-deep-dives/cursor-management) survives restarts. See [solanaPortalSource](./reference/basic-components/source) for the full source API and [Pipe anatomy](./guides/basic-development/anatomy) for how the pieces fit together.
